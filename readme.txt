写再最前面
1.使用skyline框架的前提是使用者要有最起码的it素养，不是快餐式的培训学校出生（绝大多数这样的同鞋并不适合it，因为coder的思想是建立在长期基础的coder基础的熏陶之上的）。
2.skyline会为使用者提供一大堆工具，但是不会引导coder如何去使用，因为作者不愿意拘束coder们的思维。
3.最简单的东西才是最美的。skyline不会效仿一些框架做一些作者认为“没有必要”的事情，比如orm。
